The scopes are shared

Since we have a shared scope (between two directives and the controller), changing on a property, like sampleData, reflects everywhere. If we dont want that, we can use an isolated scope instead. If we do that (an empty object in testDirective.js:s return object), then

  • scope.name is seperated (name property exists in on both controller and directive scopes)
  • scope.directiveProp is only accessible on the directives scope
  • scope.sampleData is only accessible on the directives scope
  • scope.onKeyup can only be run from the directive
  • The two directives on this page has there on scope
